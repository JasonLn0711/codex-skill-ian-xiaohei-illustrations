# QA Checklist

## 必過項

- 最終圖是逐張由 `image_gen` 生成，不是程式化繪圖輸出。
- 是 16:9 橫式。
- 背景是乾淨白底。
- 有指定的吉祥物；如果使用者沒有指定，預設有小黑。
- 已先選擇思考功能，再選擇對應吉祥物。
- 吉祥物代表明確思考功能，不是為了可愛或角色擴張而出現。
- 吉祥物承擔核心動作，不只是裝飾。
- 沒有把 repo 擴張成角色宇宙。
- 沒有新增第七個吉祥物；目前角色系統限制在小黑、小拳、小藍、小綠、小紫、小黃。
- 沒有復刻舊案例構圖，而是為當前文章生成了新隱喻。
- 畫面怪誕、有創意、有意思。
- 簡潔清爽，主體不超過畫面約 60%。
- 一張圖只講一個核心結構。
- 中文標註少、短、能讀。
- 橙色只用於主路徑或箭頭。
- 紅色只用於重點、問題、提醒或結果。
- 藍色只用於補充說明、回饋或系統狀態。

## 失敗訊號

出現以下情況，重生成或局部編輯：

- 左上角有「常見坑 / Workflow / 系統架構圖 / 路線圖」等標題。
- 吉祥物像表情貼圖、可愛卡通、英雄海報或既有 IP 角色。
- 吉祥物只是新增角色，沒有對應思考功能。
- 畫面像角色宇宙設定圖，而不是文章概念配圖。
- 出現第七個未登記吉祥物。
- 畫面像 PPT、課程課件、正式流程圖。
- 元素太多、箭頭太多、節點太多。
- 文字變成大段解釋。
- 背景有紙紋、陰影、漸變、米色、噪點。
- 真實 UI 截圖或科技感介面。
- 中文錯字嚴重或標註不可讀。
- 畫面太死板，沒有荒誕隱喻。
- 和 `assets/examples/` 裡的舊案例構圖過於相似。
- 如果使用小拳，畫面出現既有漫畫、動畫、遊戲或品牌角色的可辨識服裝、臉、名稱、標誌、招式或世界觀。
- 任何使用 PIL、SVG、Canvas、HTML、Matplotlib、Graphviz、自寫繪圖腳本、截圖或流程圖工具產生的替代示例圖。

## 迭代方法

- 太普通：讓指定吉祥物成為動作主體，加入一個奇怪但成立的隱喻。
- 太複雜：刪節點，只保留一個動作和 3-5 個短標註。
- 太可愛：強調 deadpan、blank serious expression、not cute、not mascot。
- 太 PPT：去掉標題、邊框、整齊網格和過多箭頭，改成手繪場景。
- 太像舊案例：保留核心意思，換掉主物件和吉祥物動作。
- 文字錯：優先局部編輯；錯得多就重生成並減少標註數量。
- 來源錯：如果圖是程式畫出來的，不要修補；直接廢棄並逐張重跑 `image_gen`。

## 交付判斷

高品質圖應該讓讀者先覺得「有點怪」，然後 1 秒內看懂結構。

如果第一眼像教學頁，而不是白紙上的怪誕產品草圖，就不合格。
