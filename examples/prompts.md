# Prompt Examples

下面這些 prompt 可以直接複製到 Codex 裡使用。

## 只做配圖規劃

```text
Use $ian-xiaohei-illustrations 先不要生成圖片。
請分析下面這篇文章哪裡值得配圖，輸出 5 張左右的 shot list。
每張圖寫清楚：
- 放在哪個段落後
- 圖的主題
- 核心意思
- 結構類型
- 小黑在圖裡做什麼
- 建議元素
- 建議中文標註詞

<貼上文章>
```

## 文章內文配圖

```text
Use $ian-xiaohei-illustrations 把下面這篇文章生成 4 張小黑怪誕內文配圖。
要求：16:9 橫式、純白背景、黑色手繪線稿、少量紅橙藍中文手寫批註。
每張圖只講一個核心結構，不要做 PPT 資訊圖，不要可愛卡通。

<貼上文章>
```

## 長文配圖策略

```text
Use $ian-xiaohei-illustrations 給這篇長文做配圖策略。
不要平均配圖，只挑認知錨點：核心判斷、輸入輸出閉環、前後對比、常見坑、承接路徑。
預設 6-8 張，先輸出 shot list，不要生成圖片。

<貼上文章>
```

## 單個觀點生成一張圖

```text
Use $ian-xiaohei-illustrations 為這個觀點生成一張 16:9 內文配圖：

信任不是喊出來的，而是一塊證據一塊證據鋪過去。

畫面要怪誕但清爽，小黑必須承擔核心動作。
中文標註最多 5 個，短一點。
```

## 工作流程主題

```text
Use $ian-xiaohei-illustrations 為「把一條原始素材加工成流量、信任、轉換三種內容」生成一張圖。
不要畫正式流程圖，不要復刻一魚多吃舊案例。
請重新發明一個新的低科技隱喻，讓小黑參與核心動作。
```

## 改圖：去掉標題

```text
Use $ian-xiaohei-illustrations 幫我編輯這張圖。
去掉左上角的「Workflow / 流程圖」標題和底線，其他內容保持不變。
不要新增任何文字或物件。
```

## 改圖：增強小黑參與感

```text
Use $ian-xiaohei-illustrations 這張圖方向對，但小黑有點像裝飾。
請保持核心意思不變，重生成一版：讓小黑成為真正推動結構運轉的人。
畫面更怪一點，但仍然純白、清爽、少字。
```

## 生成一組風格樣片

```text
Use $ian-xiaohei-illustrations 輸出 5 個不同主題的小黑內文配圖效果。
主題分別覆蓋：資訊過載、產品驗證、內容複利、一人公司、信任建立。
每張單獨生成，不要拼成一張。
```
